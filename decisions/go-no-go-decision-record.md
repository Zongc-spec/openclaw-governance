# Go / No-Go Decision Record v1

## Purpose
Provide a single canonical decision record for activation judgment.

## Decision Options
- NO_GO
- CONDITIONAL_NO_GO
- DEFER
- GO_TO_NEXT_REVIEW_STEP

## Required Fields
- decision_date
- decision_maker
- current_state
- decision
- rationale
- blocking_gaps
- next_required_action
- reviewed_artifacts

## Core Rule
No activation may proceed without an explicit written decision record.
