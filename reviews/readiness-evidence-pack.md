# Readiness Evidence Pack v1

## Purpose
Collect the minimum evidence required for a future activation review.

## Required Evidence
- protected branch governance active
- constitution files present
- policy files present
- runtime control specs present
- activation gate spec present
- audit schema present
- operator control surface spec present
- emergency freeze spec present
- request expiry spec present

## Evidence Rule
Each evidence item must be referenceable by file path or repository state.

## Non-Evidence
The following do not count as activation evidence by themselves:
- casual chat intent
- verbal assumption
- undocumented runtime behavior
- unreviewed local experiments
