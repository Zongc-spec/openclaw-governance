# Runtime Evidence Plan v1

## Purpose
Define what implementation evidence must exist before activation can be reconsidered.

## Required Evidence Areas
1. command parsing evidence
2. normalization evidence
3. actor verification evidence
4. approval handling evidence
5. pending request store evidence
6. audit record emission evidence
7. STOP priority evidence
8. GLOBAL_FREEZE evidence
9. protected-target blocking evidence
10. read-only fallback evidence
11. failure-path evidence

## Evidence Standard
Evidence should be reproducible, reviewable, and tied to specific implementation artifacts and tests.

## Non-Evidence
The following do not qualify:
- verbal confirmation
- unrecorded manual checks
- "it worked once" reports
- undocumented local behavior
