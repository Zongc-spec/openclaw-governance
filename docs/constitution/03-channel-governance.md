# Channel Governance

## Approved Channel State
No execution channel is approved in Phase 1.
Discord may be prepared in later phases but is not yet authorized for execution.

## Channel Function
A channel may be used for:
- command intake
- status request
- approval reply
- incident notification
- handoff summary

## Channel Limits
A channel must not be treated as:
- policy source of truth
- credential storage
- permanent audit archive
- constitutional amendment surface

## Allowed Command Types
- STATUS
- READ
- PLAN
- DRAFT
- PROPOSE_CHANGE
- APPROVE_CHANGE
- REJECT_CHANGE
- INCIDENT
- STOP

## Forbidden Channel Behaviors
- executing ambiguous destructive commands
- accepting approvals from unverified identity
- taking action from edited or partial commands without revalidation
- treating channel memory as durable policy
