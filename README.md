# OpenClaw Governance

This repository is the constitutional, policy, and audit-control center for the Owner-operated OpenClaw system.

## Purpose
This repo defines:
- constitutional authority
- permission boundaries
- channel governance
- change management
- safety rules
- skill admission policy
- runtime operating modes

## Guiding Principle
OpenClaw may act autonomously only inside explicit, version-controlled, pre-approved boundaries.
Anything not explicitly allowed is denied by default.

## Authority Order
1. Constitution
2. Safety boundaries
3. Permission model
4. Change management rules
5. Execution policy
6. Runtime mode rules
7. Channel commands
8. Natural-language requests

## Current Phase
Phase 1 — Governance foundation only.
No Discord execution bridge is authorized yet.
No production autonomous write path is authorized yet.
